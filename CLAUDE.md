# tans

Tangram-themed habit tracker for iOS. Each day you complete a habit, you earn one of seven tangram pieces; seven days fills out the week's silhouette.

This is the **Expo + React Native** rewrite of [`../habit-loop`](../habit-loop) (a Vite/React/shadcn web app generated by Lovable). The web repo is reference-only — no shared code at runtime, but the domain logic, types, and silhouette data were copy-ported.

## Stack

- **Expo SDK 54** + **React Native 0.81** + **React 19** (new arch enabled)
- **expo-router** v6 — single screen at [app/index.tsx](app/index.tsx)
- **react-native-reanimated 4** + **Moti** for animations
- **react-native-svg** for the tangram board
- **@react-native-async-storage/async-storage** — single key, schema-versioned
- **expo-notifications** — daily reminder + evening check-in (foreground handler installed)
- **expo-haptics** — `Success` impact on habit completion
- **@expo-google-fonts/fredoka** + **@expo-google-fonts/space-grotesk** — display + body fonts
- **@expo/vector-icons** (Ionicons) for icons (replaces lucide-react from web)
- **pnpm** (NOT bun — user is new to bun and we kept the new project on the lower-friction tooling)

No backend, no auth, no sync. All state local.

## Layout

```
app/
  _layout.tsx        # font loading + notification setup; hides splash when ready
  index.tsx          # the single screen (everything: header, tangram board, habit list, sheets, FAB)
components/
  Tangram.tsx        # 7-piece SVG board
  FallingTangram.tsx # celebratory overlay (Moti spring drop) on habit completion
  EveningCheckIn.tsx # ≥7pm panel: 1-10 likelihood + next-habit idea
  MissedDaySheet.tsx # bottom sheet: "be gentle" badge picker / small-win textarea
constants/
  theme.ts           # colors (brick palette), radius scale, font names — light mode only
lib/
  types.ts           # Habit, DayEntry, EveningCheckIn, MissedDay, AppState (ports of habit-loop)
  date.ts            # getToday, getWeekStart, getDaysOfWeek, isEvening — local time, NOT UTC
  silhouettes.ts     # 5 weekly silhouettes + getSilhouetteForWeek hash
  store.ts           # useHabitStore — AsyncStorage-backed, async hydration
  notifications.ts   # setupNotifications: permission + daily DAILY triggers
```

Path alias `@/*` → repo root ([tsconfig.json](tsconfig.json)).

## Persistence model

- Single AsyncStorage key: `tans:state:v1`.
- Persisted blob: `{ schemaVersion: 1, data: AppState }`. The version field is the migration hook for future schema changes — bump it and write a migration in [lib/store.ts](lib/store.ts).
- Hydration is async; the store exposes a `hydrated` boolean and skips writes until hydration finishes (otherwise we'd clobber real data with the empty default on first render).
- IDs use a non-crypto Math.random UUID v4 helper in [lib/store.ts](lib/store.ts) — fine for local-only data.

## Animation conventions

- **Moti** for everything that's a tween/spring on a `View`. Pattern:
  ```tsx
  <MotiView from={{ translateY: 600 }} animate={{ translateY: 0 }} exit={{ translateY: 600 }}
            transition={{ type: 'spring', damping: 22, stiffness: 240 }} />
  ```
- **AnimatePresence** from `moti` (NOT framer-motion) wraps conditionally-rendered transitions.
- Bottom sheets are NOT `@gorhom/bottom-sheet` — they're `MotiView` slides on a `KeyboardAvoidingView` with a `Pressable` scrim. If we ever need drag-to-dismiss, swap individual sheets to gorhom; for v1 the simpler approach is consistent.

## Notifications

[lib/notifications.ts](lib/notifications.ts) sets up two repeating local notifications:

- `tans.morning-reminder` at 09:00 — "Time to make a piece"
- `tans.evening-checkin` at 19:00 — "Evening check-in"

Both schedule via `Notifications.SchedulableTriggerInputTypes.DAILY`. Calling `setupNotifications()` is idempotent — it cancels by ID and re-schedules. Triggered from `_layout.tsx` on mount; permission denial is silent (we just don't schedule).

Times are exported as `NOTIFICATION_TIMES` and a future settings screen should read/write them.

## Conventions / gotchas

- StyleSheet + theme.ts only — no NativeWind, no Tamagui.
- Apostrophes in `<Text>` content must be the typographic `’` (U+2019), not `'` — the `react/no-unescaped-entities` lint rule fires on the straight one. Bonus: it looks better.
- `getToday()` uses local time, not UTC (deliberate fix from the web version).
- Reanimated must be imported once at app boot — `import 'react-native-reanimated'` in [_layout.tsx](app/_layout.tsx).
- `@react-navigation/*`, `expo-image`, `expo-symbols`, `expo-web-browser`, `expo-linking`, `expo-system-ui` are still in `package.json` from the Expo template but unused. Trim opportunistically once the app has stabilized.
